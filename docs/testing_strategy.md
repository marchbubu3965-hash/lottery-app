# 抽籤系統測試策略（Testing Strategy）

本文件說明本專案「抽籤系統」的測試設計原則、測試層級劃分，以及各類測試所覆蓋的行為與責任邊界。

---

## 一、測試目標（Testing Goals）

本專案的測試設計目標如下：

1. **確保核心抽籤流程正確**
   - 獎項依序抽取
   - 狀態轉換合法且可預期
2. **確保特別獎流程正確**
   - 僅產生一名得獎者
   - 得獎者來源正確
3. **避免 UI 與業務邏輯耦合**
   - 所有核心邏輯可在無 UI 的情況下測試
4. **提升未來維護與重構的安全性**

---
